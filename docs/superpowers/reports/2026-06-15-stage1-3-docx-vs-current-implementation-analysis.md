## 2. 三阶段产品路线复述

### 2.1 阶段一：情绪创可贴与心理疗愈

阶段一产品核心不是普通陪聊工具，也不是“复活逝者”的数字人，而是一个数字哀伤辅导工具。它的底线原则是克制、安全、边界清晰，帮助失亲者和极度孤独者完成情绪安放与逐步回到现实。

阶段一关键模块：

- 极低门槛记忆提取器：旧照片、微信语音、截图、口头禅、性格描述。
- 时空信箱：延迟投递与回声，避免即时聊天造成沉迷。
- 向阳生长：渐进式脱敏、对话时长限制、危机词监控、心理干预热线。
- 适老模式：一键对话、隐形健康雷达、子女脱敏周报。
- 一个大脑，两副面孔：底层星桥引擎通用，前端 A 是勿忘 App，前端 B 是适老终端或小程序。

### 2.2 阶段二：数字镜像与家庭星桥

阶段二从“疗愈逝去”转向“赋能当下”。核心思想是用高频低阻力工具形态无痛采集日常数据，形成个人数字副驾和家庭情感枢纽。

阶段二关键模块：

- 私人专属 Copilot：语音、文字、拍照备忘、吐槽、求助，AI 通过自然追问萃取价值观和长期记忆。
- 星桥家庭空间：家庭成员的数字分身或影像档案，支持异步留言/视频通信。
- 数据防火墙：私人 Copilot 的吐槽绝对保密，只有用户确认后的“个人记忆折页”才能进入家庭空间。
- 封箱仪式：亲人重病或离世后进入静默沉睡，再通过星辰化仪式切换到阶段一哀伤辅导逻辑。
- 异步跨时空同频：例如父亲随口语音扩写成孩子专属睡前故事。

### 2.3 阶段三：永恒守望与精神遗产

阶段三是终极形态，从日常陪伴和数据积累升维为数字资产管理、精神遗产托管和元宇宙智能体社会。

阶段三关键模块：

- 星桥永恒信托库：私有化部署的数字董事会、深度 RAG、专属大模型、2.5D/3D 数字人终端。
- 数字遗嘱分级控制系统：公开域、家族域、高密域，支持条件触发、身份验证和分级授权。
- 星桥永恒元宇宙社会：公众人物或家族授权的数字 Agent 入驻空间计算社区，并形成自主交互社会。
- 顶奢定制与托管：数字人传记工程、年度托管、虚拟咨询和 IP 授权。

## 5. 阶段二目标与当前实现对照

阶段二是从“失亲疗愈”扩展到“活人日常数字副驾 + 家庭情感空间”。当前工程有一些可复用基础，但还没有阶段二产品闭环。

### 5.1 私人专属 Copilot

| 阶段二目标 | 当前相关能力 | 差距 |
| --- | --- | --- |
| 随手语音、文字、拍照备忘 | 首页语音、档案文本/照片、KBLite 入库。 | 输入仍围绕阶段一记忆档案和数字人对话，不是通用个人副驾。 |
| 自然追问萃取价值观 | 数字人 prompt 有追问原则，KBLite 有知识缺口。 | 缺价值观 schema、人生决策模型、长期复盘面板。 |
| 个人数字孪生知识库 | KBLite 是轻量知识图谱，已有隐私裁剪和后端快照。 | 当前偏人物/地点/事件/事实，不足以表达职业偏好、家庭关系张力、价值观和决策模式。 |
| 客观镜子/数字董事会雏形 | 无明确产品页面。 | 需要新增个人 Copilot 信息架构和决策建议边界。 |

判断：阶段二 Copilot 可复用当前首页语音、KBLite、隐私分层和后端，但必须扩展数据模型和交互目标，不能直接把阶段一数字人聊天换个名字。

### 5.2 星桥家庭空间

| 阶段二目标 | 当前相关能力 | 差距 |
| --- | --- | --- |
| 家庭情感客厅 | 亲友、家族知识库同步、足迹、关怀看板。 | 没有统一“家庭空间”视觉与信息架构。 |
| 高质量 2.5D 动态数字分身 | 当前有单数字人资源和 TTS/口型链路。 | 没有多家庭成员分身、生命周期状态、身份绑定。 |
| 异步留言/视频通信 | 时空信箱和本地信件可复用。 | 尚未扩展为家庭成员间异步通信。 |
| 家庭记忆折页授权 | 有 privacy scope、familyCircle、sourceRef。 | 缺“每周记忆折页 -> 用户确认 -> 同步家庭空间”的产品机制。 |

判断：家庭空间的底层素材和权限已有雏形，但产品层尚未建立。阶段二设计应优先做数据防火墙和授权折页，而不是先做炫酷家庭 3D 空间。

### 5.3 封箱仪式与生死边界

| 阶段二目标 | 当前相关能力 | 差距 |
| --- | --- | --- |
| 数据源断联后静默沉睡 | 未见完整生命周期状态机。 | 需要 family member/person profile 的状态：active、inactive、sleeping、memorialized 等。 |
| 长按头像触发封存记忆仪式 | 当前无封箱仪式。 | 需要产品设计、权限确认、心理边界文案、不可逆/可逆策略。 |
| 自动切换阶段一哀伤辅导逻辑 | 阶段一哀伤辅导基础存在。 | 缺从阶段二日常陪伴到阶段一哀伤辅导的生命周期桥。 |

判断：封箱仪式是阶段二到阶段一/三的关键桥梁，目前没有实现。后续如果进入阶段二，必须先设计人物生命周期和授权模型。

## 6. 阶段三目标与当前实现对照

阶段三远超当前工程范围。当前工程只能提供早期数据底座，不能宣称具备数字信托或元宇宙社会能力。

### 6.1 星桥永恒信托库

| 阶段三目标 | 当前相关能力 | 差距 |
| --- | --- | --- |
| 私有化部署数字董事会 | 当前有独立后端和 Postgres，尚非私有化信托系统。 | 缺多租户、私有云部署、数据保险箱、灾备、审计。 |
| 深度 RAG 和专属模型 | KBLite 是轻量图谱，DeepSeek 代理支持抽取。 | 缺大规模材料采编、向量库、微调/专属模型、证据引用系统。 |
| 2.5D/3D 数字人终端 | 当前是 iOS 内嵌数字人资源和 TTS/口型链路。 | 缺高保真资产生产、动捕、空间端和会议室交互。 |

### 6.2 数字遗嘱与 ACL

| 阶段三目标 | 当前相关能力 | 差距 |
| --- | --- | --- |
| 公开域/家族域/高密域 | 当前有 localOnly、generationAllowed、familyCircle 等隐私 scope。 | 不等于法律级 ACL；缺条件触发、身份认证、审计和不可抵赖记录。 |
| 特定条件释放信息 | 无。 | 需要遗嘱条件引擎、身份验证、时间锁、授权审批。 |
| 虹膜/DNA 等验证 | 无。 | 超出当前 App 范围。 |

### 6.3 元宇宙 Agent 社会

当前工程没有元宇宙空间计算、Agent 自主行为、多 Agent 社交、虚拟 IP 商业化系统。现有数字人、KBLite、TTS 和后端只能作为非常早期的技术种子。

结论：阶段三现在只能进入“架构预留和数据合规设计”，不应进入功能开发。

## 10. 建议的发展路线

### 10.1 当前不要急着进入阶段二/三

阶段二和阶段三的愿景很大，但当前工程最有价值的收敛动作是完成阶段一真实验收。否则后续所有家庭空间、Copilot、数字信托都会建立在不稳定的数据底座上。

阶段一完成前不建议启动：

- 独立家庭空间大改版。
- 元宇宙/空间计算原型。
- 数字遗嘱高密域。
- 新商业化页面。
- 大规模 3D 数字人资产链。

### 10.2 阶段一收口路线

1. 完成真机验收：
   - 记忆档案馆真实文本、照片、语音。
   - 数字人已知/未知事实边界。
   - 时空信箱投递和 metadata-only。
   - 亲友关怀跨设备权限。
2. 修复真机验收中出现的 P0/P1 缺陷。
3. 后端开启长期测试配置：
   - HTTPS 正式域名。
   - API token。
   - Postgres 持久化。
   - 日志脱敏。
   - 基础备份。
4. 将路演/seed/mock 能力继续收敛到测试或显式参数，不进入真实模式。
5. 形成阶段一验收包，用作阶段二需求评审输入。

### 10.3 阶段二启动条件

建议满足以下条件后再进入阶段二产品开发：

- 阶段一 P0/P1 真机验收通过。
- 真实账号下知识库连续沉淀稳定，不再出现旧 seed/mock。
- 数字人能稳定做到“有证据才回答”。
- 后端至少支持一周以上长期测试。
- 隐私 scope 和后端脱敏经抽查可信。

阶段二第一批开发不应是做炫酷家庭空间，而应是：

1. 个人 Copilot 数据模型扩展：
   - 价值观。
   - 决策偏好。
   - 情绪触发因素。
   - 家庭关系事件。
   - 用户确认后的公开摘要。
2. 记忆折页机制：
   - 每周生成私人折页。
   - 用户手动确认。
   - 确认后进入家庭空间。
3. 人物生命周期状态机：
   - active。
   - inactive。
   - sleeping。
   - memorialized。
   - sealed。
4. 家庭空间最小版：
   - 家庭成员列表。
   - 授权记忆摘要。
   - 异步留言。
   - 不展示私人 Copilot 原文。

### 10.4 阶段三预研边界

阶段三可以开始做架构预研，但不建议进入产品实现。预研应集中在：

- 数据 ACL 模型。
- 审计日志。
- 加密存储。
- 证据引用式 RAG。
- 私有化部署拓扑。
- 法律授权与数字遗嘱边界。
- 数字人资产授权协议。

## 11. 后续验收与开发优先级

| 优先级 | 任务 | 目标 |
| --- | --- | --- |
| P0 | 阶段一真机测试 | 证明当前主链路真实可用。 |
| P0 | 修复真实测试暴露的问题 | 只修真实缺陷，不重复打磨已封板路演能力。 |
| P1 | 后端生产化最小闭环 | 支撑长期真实测试。 |
| P1 | 完善安全和隐私证据 | 避免数字哀伤产品的伦理风险。 |
| P2 | 阶段二信息架构设计 | 在阶段一数据底座稳定后启动。 |
| P3 | 阶段三架构预研 | 只做设计和约束，不做大规模开发。 |

## 12. 最终判断

当前工程的方向没有偏离三阶段产品路线，反而已经形成了后续阶段最关键的底座：记忆沉淀、隐私分层、结构化知识库、数字人表达、亲友脱敏、后端 metadata 服务。

真正的风险不是“功能太少”，而是“太早跳到阶段二/三，导致阶段一真实疗愈闭环没有被验证”。阶段一文档的核心是克制、安全和好好告别；阶段二文档的核心是活人数据主权和家庭授权；阶段三文档的核心是信托级治理和长期精神资产托管。当前工程必须先把阶段一的真实验收做扎实，才能自然扩展到阶段二和阶段三。

一句话结论：

> 当前工程已经具备阶段一真实验收的工程底座，并为阶段二/三留下了正确的数据和隐私方向；下一步应停止扩展愿景功能，集中完成阶段一 P0/P1 真机证据闭环。
