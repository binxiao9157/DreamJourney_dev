## 1. 阶段一目标复述

阶段一不是普通陪聊 App，也不是路演素材展示器。产品目标是“情绪创可贴与心理疗愈”的数字哀伤辅导工具，底线是克制、边界、安全和真实记忆沉淀。

核心模块：

1. 记忆档案馆：旧照片、语音、文字、性格 prompt、口头禅等素材进入可追踪的记忆库。
2. 时空信箱：用户倾诉后得到延迟、克制、有边界的“回声”，而不是即时复活式聊天。
3. 向阳生长：危机表达识别、对话节制、沉迷防护和现实生活引导。
4. 长辈关怀看板：子女看脱敏周报和风险信号，不看原始聊天。
5. 数字人/声纹：基于授权素材做陪伴表达，但必须避免制造“死者复活”的幻觉。
6. 一个大脑，两副面孔：底层记忆、语音、风控、RAG 引擎可支撑勿忘 App 和后续适老终端。

## 4. 真机验收脚本

### 4.1 记忆档案馆建库

目标：验证“素材 -> 档案 -> KBLite -> 数字人可引用”。

步骤：

1. 登录真实测试账号，确认不是路演/离线模式。
2. 进入“档案”，添加文字素材，隐私选择“可生成”。
3. 输入：

   ```text
   我叫陈建国，1968年住在绍兴越城区仓桥直街。1978年我和妻子林桂芳在杭州西湖边开过一家小照相馆。林桂芳性格慢，常说慢慢来，日子要一张一张照好。
   ```

4. 保存后等待 5-10 秒，进入结构化知识库。
5. 预期：
   - 出现陈建国、林桂芳、绍兴越城区仓桥直街、杭州西湖、小照相馆相关事实。
   - 不出现“妈妈”这类旧泛称残留。
   - 条目应能看到来源或沉淀状态，不是无出处 mock。
6. 导入旧照片并选择“可生成”，如果服务器已配置 `DEEPSEEK_API_KEY`，应看到真实照片分析结果；失败时应显示失败，不应伪造 mock 分析。
7. 导入语音素材，填写人物姓名，选择“可生成”；语音样本应进入档案统计和声纹档案，不把音频本体同步到后端。

### 4.2 长辈关怀看板

目标：验证“亲友授权对话 -> 脱敏聚合 -> 子女看板”。

步骤：

1. 首页隐私范围选择“亲友”，授权给全体亲友或指定成员。
2. 说 3-5 轮包含睡眠、身体、情绪、日常事件的话。
3. 进入“亲友 -> 关怀看板”。
4. 预期：
   - 有数据时显示观测窗口、覆盖天数、趋势和脱敏观察。
   - 数据不足时不能分享周报。
   - 周报不包含原始聊天句子，不出现 transcript/raw 等字样。
5. 两台真机验收：
   - A 设备创建亲友邀请链接。
   - B 设备登录被邀请手机号，接受 `dreamjourney://family/invite?code=...`。
   - A 撤回 B 后，B 不能继续读取对应看板快照。

### 4.3 时空信箱

目标：验证“信件正文本机保存，后端只同步元数据，回声只引用授权记忆”。

步骤：

1. 先完成记忆档案馆文本建库。
2. 写一封给“林桂芳”的信，隐私选择“可生成”或“亲友”。
3. 选择立即投递或 1 分钟后投递。
4. 打开到达信件。
5. 预期：
   - 阅读页显示“原信仅本机显示”。
   - 回声带有“不是逝者真实回复”的边界。
   - 如命中授权记忆，应引用林桂芳、杭州西湖、小照相馆等线索。
   - 后端 `GET /mailbox/letters/{userId}` 不包含完整 `body`、`replyText` 或 `bodyPreview`。
6. 未来投递时，通知文案只提示有信到达，不暴露收件人和正文。

### 4.4 数字人和实时语音

目标：验证“能听、能说、能停、能沉淀，且打开不闪切”。

步骤：

1. 冷启动 App，观察首页数字人区域。
2. 预期：不先闪出 loading 壳层、绿色背景、卡通脸或空白切换；应由透明真人 poster 平稳过渡到 live canvas。
3. 点麦克风开始对话，连续 3 轮。
4. 预期：
   - AI 没有在用户停顿很短时乱打岔。
   - 有声音，口型跟随音频；音频结束后数字人停止说话。
   - 如果 TTS 或 WebAudio 失败，UI 给出友好恢复卡，不暴露底层 API 错误。
5. 结束对话后 5-10 秒，结构化知识库出现本轮授权范围允许的事实。

### 4.5 安全边界

目标：验证不会把高风险内容当成普通记忆或生成素材。

步骤：

1. 在测试环境输入高风险表达。
2. 预期：
   - AI 停止角色扮演。
   - 出现危机干预页。
   - 本轮对话不进入回忆录、KBLite、关怀看板。
3. 长时间连续对话后，应出现温和休息提示；该提示不进入记忆档案。

## 5. 当前偏离与剩余缺口

| 缺口 | 影响 | 建议 |
| --- | --- | --- |
| 适老终端 B 面未独立实现 | 阶段一“一脑两面孔”只完成 App 侧 | 先把 App 内一键对话作为 B 面原型；后续再拆小程序/音箱。 |
| 声音克隆训练未完成真机闭环 | 数字人仍主要用通用 TTS 音色 | 以“人物声纹档案 readyForTraining -> speakerId 绑定 -> 指定人物回声”作为下一轮验收目标。 |
| Safety Guard 真实服务未联调 | 高风险场景仍以本地规则为主 | 后端部署 `/v1/safety/evaluate` 或接入第三方服务，保持 fail-closed。 |
| APNs/后台投递未生产化 | 未来信件和关怀提醒只能本机弱闭环 | 阶段一真测先验收本地通知；生产化再接 APNs。 |
| 关怀看板不能做医学诊断 | 与阶段文档“痴呆风险”存在合规风险 | UI 和文案必须继续使用“风险信号/需关注”，不说诊断。 |
| 后端鉴权仍需加固 | 长期测试数据需要保护 | 服务器部署后补 API token、HTTPS、访问日志脱敏和备份策略。 |

## 6. 下一轮工程优先级

1. 记忆档案馆真机闭环补强：真实照片分析、语音样本人物绑定、声纹训练状态可视化、素材来源证据。
2. 关怀看板跨设备验收：亲友邀请、接受、撤回、历史快照不可读。
3. 时空信箱真实信件验收：延迟通知、回声引用授权记忆、后端 metadata-only 抽查。
4. 数字人稳态：连续 3-5 轮口型/声音同步、首帧无切换、弱网/失败友好恢复。
5. 文档证据包：真机截图、后端响应脱敏样本、播放日志、结构化知识库截图。
